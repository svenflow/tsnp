Familiar Syntax

Write what you already know

import { createBackend } from 'numpyjs';
const np = await createBackend('js');

// Create arrays
const a = np.array([[1, 2], [3, 4]]);
const b = np.eye(2);

// Linear algebra
const c = np.matmul(a, b);
const { values, vectors } = np.eig(a);

// Statistics
console.log(np.mean(a, 0));
console.log(np.std(a));
import numpy as np

# Create arrays
a = np.array([[1, 2], [3, 4]])
b = np.eye(2)

# Linear algebra
c = np.matmul(a, b)
vals, vecs = np.linalg.eig(a)

# Statistics
print(np.mean(a, axis=0))
print(np.std(a))
Note: numpyjs uses flat namespaces — np.eig() instead of np.linalg.eig(), np.randn() instead of np.random.randn(). Shape args use arrays: np.randn([3, 3]).
// Full TypeScript — types flow through every operation
const a: NDArray = np.array([[1, 2], [3, 4]]);       // NDArray
const mean: number = np.mean(a) as number;            // number
const { values, vectors } = np.eig(a);               // typed decomposition
const b: NDArray = np.randn([3, 3]);                  // NDArray

Use Cases

Where numpyjs shines

Client-side ML inference
Run tensor operations directly in the browser. Pre/post-process data for ONNX or TFLite models without a server round-trip. WebGPU backend handles large batches.
Interactive data viz
Compute statistics, histograms, and transforms on the fly. Pair with D3 or Observable Plot. FFT for audio visualizations. All at 60fps with the WASM backend.
Scientific computing in Node.js
Signal processing, matrix decomposition, and numerical methods in server-side JavaScript. Same API whether you prototype in Python or ship in JS.
Education & notebooks
Teach linear algebra and statistics in JavaScript notebooks. Students familiar with NumPy tutorials can follow along directly. Full TypeScript intellisense guides exploration.
Performance

Three backends, one API

createBackend('js') → createBackend('webgpu') — one line change, same code.
Pure JS
Zero dependencies, zero compilation. Works everywhere JavaScript runs — browsers, Node.js, Bun, Deno, edge functions. The universal default.
Node.js · Bun · All browsers
WebGPU
GPU-accelerated computation via WebGPU shaders. Massive speedups for large matrix operations, convolutions, and batch processing. Runs on the GPU.
Chrome 113+ · Edge 113+ · Firefox Nightly
WASM
Rust compiled to WebAssembly. Near-native performance for CPU-bound work. SIMD-optimized linear algebra, random number generation, and reduction operations.
All modern browsers · Node.js · Bun

Get Started

Up and running in 30 seconds

1
Install
npm install numpyjs
2
Import and create a backend
import { createBackend } from 'numpyjs';

// Pick your backend
const np = await createBackend('js');      // CPU — works everywhere
// const np = await createBackend('webgpu'); // GPU — Chrome, Edge
// const np = await createBackend('wasm');   // Rust WASM — near-native
3
Compute
const x = np.linspace(0, 2 * Math.PI, 100);
const y = np.sin(x);

const A = np.randn([3, 3]);
const { u, s, vt } = np.svd(A);

console.log('mean:', np.mean(y));
console.log('singular values:', s.toArray());
